Hexo和GitHub搭建个人静态博客

2018/02/21

搭建过程大概需要这么几个步骤：

1.搭建环境的准备工作（主要是node.js和git环境，以及GitHub账户的设置）

1、安装npm 
2、本地Git环境搭建
3、GitHub账号注册
4、配置github，创建New respository仓库，选择public
配置setting的github pages ，有个按钮 Choose a theme 选择一个主题后保存基本上初页就完成了。可以访问如：https://raochuan.github.io/

2.安装并初始化Hexo
为什么要装hexo,使博客有漂亮的皮肤，使用本地环境编译和快速发布Markdown编辑后的文档

1、新建文件夹 mkdir ‘hexo-page’
2、安装hexo,npm install hexo-cli -g 请科学上网或者改镜像
3、保存项目 npm install hexo --save
4、检测hexo安装 hexo -v
5、初始化本地脚手架 hexo -init，npm install更新框架
6、编译和本地调试 hexo g /hexo -s
本地可以调试和浏览

3.将Hexo和GitHub Page联系起来
主要是关联SSH，配置SSH keys

1、非第一次使用git生成ssh,用ssh-keygen -t rsa -C "邮件地址@youremail.com"在命令行执行
2、Mac前往文件夹 ~/.ssh/找到id_rsa.pub文本打开
3、赋值keys到github的setting里面的ssh and GPG keys
4、检测是否SSH配置成功 ssh -T git@github.com，未成功需要配置git config全局的邮箱账号
5、找到本地项目的_config.yml配置如下

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
    type: git
    repo: git@github.com:raochuan/raochuan.github.io.git
    branch: master

4.如何发布新文章

1、hexo new post "article title"创建新的.MD文件
2、在生成和部署文章前安装插件npm install hexo-deployer-git --save 避免连接不上github
3、生成:hexo g / 部署:hexo d

5.如何解析域名到自己的博客

6.Hexo插件的添加更换皮肤
1、具体可以看hexo 官方网站更换皮肤
2、需要本地安装gulp环境，具体请看gulp文档
3、点开theme主题，查看安装文档（本人使用的是hexo-theme-snippet），在项目根目录git clone 后copy gulpfile.js文件到根目录。需要注意的是要根据gulpfile里面文件依赖的库，并npm install安装一下。其中的使用gulp打包的时候报错:

Gulp Error: Cannot find module 'jshint/src/cli'
解决方案:使用npm install --save-dev jshint gulp-jshint
而不是npm install --save-dev gulp-jshint

4、安装完成注意修改_config.yml配置theme文件

# Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
**==theme: snippet==**

5、启动本地服务浏览没问题按照前面步骤生成和发布，特别注意：发布后需要安装依赖库–save,本地测试环境需要的库可以–save-dev使用

6、过程遇到的小姿势

    1、每次部署hexo d的时候要输入密码，提示:Enter passphrase for key;
    解决方案:
        eval `ssh-agent` 
        ssh-add
ssh-agent是用于管理密钥，ssh-add用于将密钥加入到ssh-agent中，SSH可以和ssh-agent通信获取密钥，这样就不需要用户手工输入密码了。 顺序执行以上两条命令后就可以用ssh免密登录远程机器了，但这个配置只对当前会话生效，会话关闭或机器重启后都需要重新执行这两条命令。将命令放到~/.bash_profile中，就可以免去每次输入的麻烦。
